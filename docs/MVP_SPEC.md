# Recipe Share MVP 기능 명세서

> 작성일: 2026-02-08
> 버전: 1.0

---

## 개요

### MVP 목표
사용자가 레시피를 등록하고, 다른 사용자의 레시피를 탐색할 수 있는 기본 서비스 구현

### 우선순위 정의
| 우선순위 | 설명 |
|----------|------|
| **P0** | 필수 - 이것 없이는 서비스 불가 |
| **P1** | 중요 - MVP에 포함되어야 함 |
| **P2** | 권장 - 시간 여유 시 구현 |

## 1. 인증 (Authentication)

### US-001: 회원가입
| 항목 | 내용 |
|------|------|
| **우선순위** | P0 |
| **예상 시간** | Backend 2h / Frontend 3h |
| **사용자 스토리** | 사용자로서, 이메일과 비밀번호로 회원가입할 수 있다. 그래서 나만의 레시피를 등록할 수 있다. |
| **인수 조건** | - 이메일 형식 검증<br>- 비밀번호 6자 이상<br>- 닉네임 2자 이상<br>- 중복 이메일 가입 불가<br>- 가입 완료 시 자동 로그인 |
| **API** | `POST /api/auth/register` |

---

### US-002: 로그인
| 항목 | 내용 |
|------|------|
| **우선순위** | P0 |
| **예상 시간** | Backend 1h / Frontend 2h |
| **사용자 스토리** | 사용자로서, 이메일과 비밀번호로 로그인할 수 있다. 그래서 내 계정에 접근할 수 있다. |
| **인수 조건** | - 이메일/비밀번호 일치 확인<br>- JWT 토큰 발급<br>- 로그인 상태 유지 (7일)<br>- 잘못된 정보 시 에러 메시지 |
| **API** | `POST /api/auth/login` |

## 2. 레시피 (Recipes)

### US-101: 레시피 목록 조회
| 항목 | 내용 |
|------|------|
| **우선순위** | P0 |
| **예상 시간** | Backend 2h / Frontend 4h |
| **사용자 스토리** | 방문자로서, 등록된 레시피 목록을 볼 수 있다. 그래서 원하는 레시피를 찾을 수 있다. |
| **인수 조건** | - 최신순 정렬<br>- 썸네일, 제목, 작성자 표시<br>- 좋아요/댓글 수 표시<br>- 페이지네이션 (12개씩)<br>- 비로그인도 접근 가능 |
| **API** | `GET /api/recipes` |

---

### US-102: 레시피 상세 조회
| 항목 | 내용 |
|------|------|
| **우선순위** | P0 |
| **예상 시간** | Backend 1h / Frontend 4h |
| **사용자 스토리** | 방문자로서, 레시피 상세 정보를 볼 수 있다. 그래서 요리를 따라할 수 있다. |
| **인수 조건** | - 제목, 설명 표시<br>- 재료 목록 표시<br>- 조리 순서 표시<br>- 조리시간, 인분, 난이도 표시<br>- 이미지 표시<br>- 작성자 정보 표시<br>- 좋아요/댓글 표시 |
| **API** | `GET /api/recipes/:id` |

---

### US-103: 레시피 등록
| 항목 | 내용 |
|------|------|
| **우선순위** | P0 |
| **예상 시간** | Backend 2h / Frontend 5h |
| **사용자 스토리** | 로그인한 사용자로서, 새 레시피를 등록할 수 있다. 그래서 나의 요리법을 공유할 수 있다. |
| **인수 조건** | - 제목 필수 입력<br>- 재료 목록 필수 (1개 이상)<br>- 조리 순서 필수 (1단계 이상)<br>- 설명, 조리시간, 인분, 난이도 선택<br>- 이미지 업로드 선택<br>- 등록 후 상세 페이지 이동 |
| **API** | `POST /api/recipes` |

**입력 필드:**
| 필드 | 타입 | 필수 | 설명 |
|------|------|------|------|
| title | string | O | 레시피 제목 |
| description | string | X | 간단한 설명 |
| ingredients | string[] | O | 재료 목록 |
| instructions | string[] | O | 조리 순서 |
| cookingTime | number | X | 조리 시간 (분) |
| servings | number | X | 인분 |
| difficulty | enum | X | easy/medium/hard |
| imageUrl | string | X | 이미지 URL |

---

### US-104: 레시피 수정
| 항목 | 내용 |
|------|------|
| **우선순위** | P1 |
| **예상 시간** | Backend 1h / Frontend 2h |
| **사용자 스토리** | 레시피 작성자로서, 내 레시피를 수정할 수 있다. 그래서 오류를 수정하거나 내용을 보완할 수 있다. |
| **인수 조건** | - 본인 레시피만 수정 가능<br>- 모든 필드 수정 가능<br>- 수정 후 상세 페이지 이동<br>- 권한 없으면 403 에러 |
| **API** | `PUT /api/recipes/:id` |

---

### US-105: 레시피 삭제
| 항목 | 내용 |
|------|------|
| **우선순위** | P1 |
| **예상 시간** | Backend 0.5h / Frontend 1h |
| **사용자 스토리** | 레시피 작성자로서, 내 레시피를 삭제할 수 있다. 그래서 더 이상 공유하고 싶지 않은 레시피를 제거할 수 있다. |
| **인수 조건** | - 본인 레시피만 삭제 가능<br>- 삭제 확인 모달<br>- 삭제 시 관련 좋아요/댓글도 삭제<br>- 삭제 후 목록 페이지 이동 |
| **API** | `DELETE /api/recipes/:id` |

---

### US-106: 레시피 검색
| 항목 | 내용 |
|------|------|
| **우선순위** | P2 |
| **예상 시간** | Backend 1h / Frontend 2h |
| **사용자 스토리** | 방문자로서, 키워드로 레시피를 검색할 수 있다. 그래서 원하는 레시피를 빠르게 찾을 수 있다. |
| **인수 조건** | - 제목, 설명에서 검색<br>- 검색 결과 개수 표시<br>- 결과 없으면 안내 메시지 |
| **API** | `GET /api/recipes?search=keyword` |

## 3. 상호작용 (Interaction)

### US-201: 좋아요
| 항목 | 내용 |
|------|------|
| **우선순위** | P1 |
| **예상 시간** | Backend 1h / Frontend 1h |
| **사용자 스토리** | 로그인한 사용자로서, 마음에 드는 레시피에 좋아요를 누를 수 있다. 그래서 관심을 표현할 수 있다. |
| **인수 조건** | - 좋아요 토글 (누르면 추가, 다시 누르면 취소)<br>- 실시간 좋아요 수 반영<br>- 비로그인 시 로그인 유도<br>- 본인 레시피도 좋아요 가능 |
| **API** | `POST /api/recipes/:id/like` |

## 비기능 요구사항

| 항목 | 요구사항 |
|------|----------|
| 응답시간 | API 응답 500ms 이내 |
| 이미지 | 최대 5MB, 자동 리사이즈 (추후) |
| 보안 | 비밀번호 해시 (bcrypt), JWT 인증 |
| 브라우저 | Chrome, Safari, Firefox 최신 버전 |
| 반응형 | 모바일/태블릿/데스크톱 지원 |
